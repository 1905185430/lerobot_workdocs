# SO-101 双臂遥操作部署文档

> 适用于 LeRobot 0.5.1+ 版本 | 2026-04-16
>
> 核心结论：官方 `lerobot` 环境（conda, Python 3.12）已内置 `bi_so_follower` / `bi_so_leader` 双臂模块，无需额外安装，接好4条臂即可使用。

## 1. 硬件清单

双臂遥操作需要 **4条 SO-101 机械臂**：

| 编号 | 角色 | 类型 | 说明 |
|------|------|------|------|
| 左从臂 | Follower | 执行臂 | 左侧执行动作的机械臂 |
| 右从臂 | Follower | 执行臂 | 右侧执行动作的机械臂 |
| 左主臂 | Leader | 操控臂 | 人左手操控的主臂 |
| 右主臂 | Leader | 操控臂 | 人右手操控的主臂 |

配件：4x USB-C线 + 4x电源适配器 + 4x桌夹

## 2. 串口连接与映射

### 2.1 端口规划

| 设备 | 推荐串口 | 标定 ID |
|------|----------|---------|
| 左从臂 (Follower) | `/dev/ttyACM0` | `bimanual_follower_left` |
| 右从臂 (Follower) | `/dev/ttyACM1` | `bimanual_follower_right` |
| 左主臂 (Leader) | `/dev/ttyACM2` | `bimanual_leader_left` |
| 右主臂 (Leader) | `/dev/ttyACM3` | `bimanual_leader_right` |

> 实际端口可能因插拔顺序变化，务必用 `lerobot-find-port` 确认。

### 2.2 确认端口

```bash
# 查看所有串口设备
ls /dev/ttyACM*

# 逐个确认哪条臂对应哪个端口（需交互操作）
lerobot-find-port
# 拔掉一条臂 → 观察哪个端口消失 → 标记该臂的端口 → 插回
# 重复4次
```

### 2.3 串口权限

```bash
# 临时授权
sudo chmod 666 /dev/ttyACM*

# 永久（需重新登录生效）
sudo usermod -aG dialout $USER
```

## 3. 环境准备

```bash
conda activate lerobot

# 确认双臂模块可用
python -c "from lerobot.robots.bi_so_follower import BiSOFollower; print('bi_so_follower OK')"
python -c "from lerobot.teleoperators.bi_so_leader import BiSOLeader; print('bi_so_leader OK')"

# 确认波特率已修复为 500000（CH9101F芯片1Mbps不可靠）
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")
grep "DEFAULT_BAUDRATE" "$FEETECH_PATH"
# 应输出: DEFAULT_BAUDRATE = 500_000
```

> ⚠️ 如果波特率为 1_000_000，参考 [SO101通信失败修复记录](SO101通信失败修复记录.md) 修复。

## 4. 电机设置

> ⚠️ 如果你的臂已经在单臂模式下使用过（ID和波特率已配好），**这步可以跳过**，直接进第5节校准。首次使用新臂（出厂状态）或更换过电机时才需要执行。

4条臂各执行一次：

```bash
# 左从臂
lerobot-setup-motors --robot.type=so101_follower --robot.port=/dev/ttyACM0

# 右从臂
lerobot-setup-motors --robot.type=so101_follower --robot.port=/dev/ttyACM1

# 左主臂
lerobot-setup-motors --teleop.type=so101_leader --teleop.port=/dev/ttyACM2

# 右主臂
lerobot-setup-motors --teleop.type=so101_leader --teleop.port=/dev/ttyACM3
```

## 5. 校准

### 5.1 交互式校准（首次使用）

4条臂各校准一次，需在终端中手动操作（将臂移到中间位置 → 转动各关节 → 确认）：

```bash
# 左从臂
lerobot-calibrate \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.id=bimanual_follower_left

# 右从臂
lerobot-calibrate \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM1 \
  --robot.id=bimanual_follower_right

# 左主臂
lerobot-calibrate \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM2 \
  --teleop.id=bimanual_leader_left

# 右主臂
lerobot-calibrate \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM3 \
  --teleop.id=bimanual_leader_right
```

### 5.2 校准文件存储位置

```
~/.cache/huggingface/lerobot/calibration/
├── robots/so_follower/
│   ├── bimanual_follower_left.json
│   └── bimanual_follower_right.json
└── teleoperators/so_leader/
    ├── bimanual_leader_left.json
    └── bimanual_leader_right.json
```

### 5.3 校准值写入电机

运行遥操作时如果提示 "Mismatch between calibration values"，直接按 **Enter** 即可，程序会自动把校准文件写入电机：

```
Press ENTER to use provided calibration file associated with the id bimanual_follower_left, or type 'c' and press ENTER to run calibration:
```

- 按 **Enter** → 用已有校准文件覆盖电机（推荐，秒完成）
- 按 **c** → 重新交互式校准（需要手动转动关节）

> 也可以提前单独执行校准写入，避免遥操作启动时卡在提示：
> ```bash
> # 每条臂单独跑一次，遇到提示按 Enter
> lerobot-calibrate --robot.type=so101_follower --robot.port=/dev/ttyACM0 --robot.id=bimanual_follower_left
> lerobot-calibrate --robot.type=so101_follower --robot.port=/dev/ttyACM1 --robot.id=bimanual_follower_right
> lerobot-calibrate --teleop.type=so101_leader --teleop.port=/dev/ttyACM2 --teleop.id=bimanual_leader_left
> lerobot-calibrate --teleop.type=so101_leader --teleop.port=/dev/ttyACM3 --teleop.id=bimanual_leader_right
> ```

### 5.4 复用已有校准文件（可选）

如果左臂之前已校准过（单臂模式的 `so101_cong_left` / `so101_zhu_left`），可直接复制：

```bash
# 从臂
cp ~/.cache/huggingface/lerobot/calibration/robots/so_follower/so101_cong_left.json \
   ~/.cache/huggingface/lerobot/calibration/robots/so_follower/bimanual_follower_left.json

# 主臂
cp ~/.cache/huggingface/lerobot/calibration/teleoperators/so_leader/so101_zhu_left.json \
   ~/.cache/huggingface/lerobot/calibration/teleoperators/so_leader/bimanual_leader_left.json
```

右臂仍需单独校准。

## 6. 双臂遥操作

### 6.1 标准双臂遥操作

```bash
lerobot-teleoperate \
  --robot.type=bi_so_follower \
  --robot.left_arm_config.port=/dev/ttyACM0 \
  --robot.right_arm_config.port=/dev/ttyACM1 \
  --robot.id=bimanual_follower \
  --teleop.type=bi_so_leader \
  --teleop.left_arm_config.port=/dev/ttyACM2 \
  --teleop.right_arm_config.port=/dev/ttyACM3 \
  --teleop.id=bimanual_leader \
  --display_data=true \
  --fps 30
```

### 6.2 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `--robot.type` | bi_so_follower | 双臂从臂类型（内置） |
| `--robot.left_arm_config.port` | /dev/ttyACM0 | 左从臂串口 |
| `--robot.right_arm_config.port` | /dev/ttyACM1 | 右从臂串口 |
| `--robot.id` | bimanual_follower | 从臂标定ID前缀（自动拼接 _left/_right） |
| `--teleop.type` | bi_so_leader | 双臂主臂类型（内置） |
| `--teleop.left_arm_config.port` | /dev/ttyACM2 | 左主臂串口 |
| `--teleop.right_arm_config.port` | /dev/ttyACM3 | 右主臂串口 |
| `--teleop.id` | bimanual_leader | 主臂标定ID前缀（自动拼接 _left/_right） |

### 6.3 带摄像头的双臂遥操作

```bash
lerobot-teleoperate \
  --robot.type=bi_so_follower \
  --robot.left_arm_config.port=/dev/ttyACM0 \
  --robot.right_arm_config.port=/dev/ttyACM1 \
  --robot.id=bimanual_follower \
  --robot.left_arm_config.cameras='{"top": {"type": "opencv", "index_or_path": "/dev/video2", "fps": 30, "width": 640, "height": 480}}' \
  --robot.right_arm_config.cameras='{"wrist": {"type": "opencv", "index_or_path": "/dev/video10", "fps": 30, "width": 640, "height": 480}}' \
  --teleop.type=bi_so_leader \
  --teleop.left_arm_config.port=/dev/ttyACM2 \
  --teleop.right_arm_config.port=/dev/ttyACM3 \
  --teleop.id=bimanual_leader \
  --display_data=true \
  --fps 30
```

> 摄像头名称会自动加 `left_` / `right_` 前缀，如 `left_top`、`right_wrist`。

## 8. 架构说明

双臂模块基于单臂模块封装，核心逻辑：

```
BiSOFollower (bi_so_follower)
├── left_arm: SOFollower  → 观测/动作前缀 left_
└── right_arm: SOFollower → 观测/动作前缀 right_

BiSOLeader (bi_so_leader)
├── left_arm: SOLeader  → 动作前缀 left_
└── right_arm: SOLeader → 动作前缀 right_
```

- 标定ID：`--robot.id=bimanual_follower` → 自动查找 `bimanual_follower_left` 和 `bimanual_follower_right`
- 摄像头：各臂独立配置，观测key自动加臂前缀
- 源码路径（PyPI安装）：
  - `lerobot/robots/bi_so_follower/bi_so_follower.py`
  - `lerobot/robots/bi_so_follower/config_bi_so_follower.py`
  - `lerobot/teleoperators/bi_so_leader/bi_so_leader.py`
  - `lerobot/teleoperators/bi_so_leader/config_bi_so_leader.py`

## 9. 常见问题

### Q1: 端口号ttyACM0/1/2/3对不上

拔插USB后端口号可能变化。解决：
- 每次启动前用 `ls /dev/ttyACM*` 确认
- 用 `lerobot-find-port` 逐个排查
- 或设置 udev 规则固定端口名（进阶方案）

### Q2: "Mismatch between calibration values" 校准不匹配

参考 5.3 节手动写入校准值，或删除对应校准文件重新校准：
```bash
rm ~/.cache/huggingface/lerobot/calibration/robots/so_follower/bimanual_follower_right.json
# 然后重新执行 5.1 节的校准命令
```

### Q3: 一条臂过载导致整组掉线

双臂模式下一条臂过载不会影响另一条臂，但该臂会掉线。恢复步骤：
1. 手动将掉线臂移到安全位置
2. 重新插拔该臂USB线
3. `sudo chmod 666 /dev/ttyACM*`
4. 用 5.3 节脚本重写校准值
5. 重新启动遥操作

### Q4: 校准文件找不到 / no calibration file found

检查 `--robot.id` 和 `--teleop.id` 是否正确。BiSOFollower 会自动拼接 `_left` / `_right` 后缀：
- `--robot.id=bimanual_follower` → 查找 `bimanual_follower_left.json` 和 `bimanual_follower_right.json`
- 如果只存在 `so101_cong_left.json`，需要复制或重新校准

### Q5: "command not found: lerobot-teleoperate"

```bash
conda activate lerobot
# 如果仍然找不到
cd ~/lerobot && pip install -e ".[feetech]"
```

### Q6: pip升级lerobot后双臂连不上

升级会覆盖所有本地修改（包括波特率修复）。需要：
1. 重新修改 `DEFAULT_BAUDRATE = 500_000`（参考 [SO101通信失败修复记录](SO101通信失败修复记录.md)）
2. 重新写入校准值

## 10. 一键启动脚本

保存为 `~/teleop_bimanual.sh`：

```bash
#!/bin/bash
# SO-101 双臂遥操作一键启动脚本

# 串口权限
sudo chmod 666 /dev/ttyACM* 2>/dev/null

# 激活环境
source ~/anaconda3/etc/profile.d/conda.sh
conda activate lerobot

# 检查波特率
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")
if grep -q "DEFAULT_BAUDRATE = 1_000_000" "$FEETECH_PATH" 2>/dev/null; then
    echo "⚠️  检测到波特率为 1Mbps，正在修复为 500000..."
    sed -i 's/DEFAULT_BAUDRATE = 1_000_000/DEFAULT_BAUDRATE = 500_000/' "$FEETECH_PATH"
    echo "✅ 波特率已修复"
fi

# 启动双臂遥操作
lerobot-teleoperate \
  --robot.type=bi_so_follower \
  --robot.left_arm_config.port=/dev/ttyACM0 \
  --robot.right_arm_config.port=/dev/ttyACM1 \
  --robot.id=bimanual_follower \
  --teleop.type=bi_so_leader \
  --teleop.left_arm_config.port=/dev/ttyACM2 \
  --teleop.right_arm_config.port=/dev/ttyACM3 \
  --teleop.id=bimanual_leader \
  --display_data=true \
  --fps 30
```

```bash
chmod +x ~/teleop_bimanual.sh
./teleop_bimanual.sh
```
