## 一、程序概览

本方案包含两个独立程序目录：

```
SO101_calibrate/                  # 校准与设备识别工具集
├── so101_cong_left.py           # 左从臂校准
├── so101_cong_right.py          # 右从臂校准
├── so101_zhu_left.py            # 左主臂校准
├── so101_zhu_right.py           # 右主臂校准
├── camera_identifier.py         # 相机识别 GUI 工具
├── camera_names.txt             # 相机预设名称配置
├── USB端口映射表.md              # 硬件端口映射文档
├── camera_mapping.json          # (运行后生成) 相机映射结果
├── camera_mapping.md            # (运行后生成) 相机映射文档
└── 99-so101-cameras.rules       # (运行后生成) udev 规则

SO101_teleoperate/                # 遥操作主程序
└── teleoperate.py               # 三种遥操作模式统一入口
```

## 二、环境准备

```bash
conda activate lerobot   # 必须用官方lerobot环境，不能用lerobot-seeed

# 确认波特率已修复为 500000（CH9101F芯片1Mbps不可靠）
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")
grep "DEFAULT_BAUDRATE" "$FEETECH_PATH"
# 应输出: DEFAULT_BAUDRATE = 500_000
# 如不是，执行：
# sed -i 's/DEFAULT_BAUDRATE = 1_000_000/DEFAULT_BAUDRATE = 500_000/' "$FEETECH_PATH"
```

> ⚠️ 每次 pip install/upgrade lerobot 后需重新修改波特率！

## 三、校准工具 (SO101_calibrate/)

### 3.1 机械臂校准

4条臂首次使用需逐个校准。每个脚本会自动申请串口权限（可能需要输入 sudo 密码），然后进入交互式校准流程。

```bash
# 从臂校准
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/so101_cong_left.py
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/so101_cong_right.py

# 主臂校准
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/so101_zhu_left.py
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/so101_zhu_right.py
```

校准步骤（按终端提示操作）：
1. 将机械臂移到行程中间位置，按 Enter
2. 依次转动各关节通过完整运动范围
3. 按 Enter 完成记录

校准文件保存位置：
```
~/.cache/huggingface/lerobot/calibration/
├── robots/so_follower/so101_cong_left.json
├── robots/so_follower/so101_cong_right.json
├── teleoperators/so_leader/so101_zhu_left.json
└── teleoperators/so_leader/so101_zhu_right.json
```

各脚本端口映射：

| 脚本 | 臂名称 | 角色 | by-id 路径 |
|---|---|---|---|
| so101_cong_left.py | so101_cong_left | follower | /dev/serial/by-id/usb-1a86_USB_Single_Serial_5B42073876-if00 |
| so101_cong_right.py | so101_cong_right | follower | /dev/serial/by-id/usb-1a86_USB_Single_Serial_5B41532613-if00 |
| so101_zhu_left.py | so101_zhu_left | leader | /dev/serial/by-id/usb-1a86_USB_Single_Serial_5B41533034-if00 |
| so101_zhu_right.py | so101_zhu_right | leader | /dev/serial/by-id/usb-1a86_USB_Single_Serial_5B42137834-if00 |

### 3.2 相机识别 GUI 工具

```bash
# 依赖安装（首次使用）
pip install PySimpleGUI opencv-python

# 运行
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/camera_identifier.py
```

功能：
- 自动检测所有 /dev/video* 设备
- 实时预览画面，确认相机身份
- 手动输入或从预设名称中选择分配
- 保存 camera_mapping.json、camera_mapping.md 和 udev 规则文件
- 预设名称列表通过 camera_names.txt 自定义（每行一个名称，# 开头为注释）

操作流程：
1. 左侧设备列表选择相机 → 右侧显示实时预览
2. 在"名称分配"栏输入名称，或点击"预设名称"快捷填入
3. 点击"保存配置" → 生成三个文件

安装 udev 规则（使相机设备名固定化）：
```bash
sudo cp /home/xuan/le_xuan/01_SO101硬件与调试/SO101_calibrate/99-so101-cameras.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger
```

## 四、遥操作程序 (SO101_teleoperate/)

### 4.1 启动方式

```bash
conda activate lerobot
python /home/xuan/le_xuan/01_SO101硬件与调试/SO101_teleoperate/teleoperate.py
```

启动后会显示菜单：

```
==================================================
      🌟 欢迎使用 SO101 遥操作控制系统 🌟
==================================================
请选择你的遥操作模式：
  [1] 一对一遥操作 (单臂控制单主从臂)
  [2] 一对多遥操作 (单主臂同步控制双从臂)
  [3] 双臂遥操作   (双主臂独立控制双从臂)
==================================================
输入选项数字 (1/2/3) 并回车:
```

### 4.2 三种模式详解

**模式 1 — 一对一遥操作**

单主臂控制对应侧的单从臂。

```
选择 1 → 左主臂 → 左从臂
选择 2 → 右主臂 → 右从臂
```

适用场景：单臂调试、单臂数据采集。

**模式 2 — 一对多遥操作**

一条主臂（默认右主臂）同步控制两条从臂，两条从臂做完全相同的动作。

```
右主臂 → 左从臂 + 右从臂（同步动作）
```

适用场景：双从臂对称任务（如双手同步抓取）、单手操控双臂演示。

**模式 3 — 双臂遥操作**

左主臂独立控制左从臂，右主臂独立控制右从臂。

```
左主臂 → 左从臂
右主臂 → 右从臂（同时独立运行）
```

适用场景：双臂协同任务（如双手交接物体）、双臂数据采集。

### 4.3 退出方式

所有模式下，按 **Ctrl+C** 安全断开连接并退出。

### 4.4 端口绑定

程序使用 /dev/serial/by-id/ 固定路径，拔插 USB 后无需重新确认端口号：

| 臂 | 类型 | by-id 路径 |
|---|---|---|
| 左从臂 | SO101Follower | usb-1a86_USB_Single_Serial_5B42073876-if00 |
| 右从臂 | SO101Follower | usb-1a86_USB_Single_Serial_5B41532613-if00 |
| 左主臂 | SO101Leader | usb-1a86_USB_Single_Serial_5B41533034-if00 |
| 右主臂 | SO101Leader | usb-1a86_USB_Single_Serial_5B42137834-if00 |

## 五、与之前方案的对比

### 5.1 总体对比

| 对比项 | 之前方案（CLI 命令行） | 本方案（Python 脚本） |
|---|---|---|
| 启动方式 | 手动输入长 CLI 命令 | 交互式菜单选择 |
| 校准 | `lerobot-calibrate` + 手动拼参数 | 独立 Python 脚本，一键运行 |
| 遥操作 | `lerobot-teleoperate` + 手动拼参数 | `python teleoperate.py` 选数字 |
| 端口指定 | /dev/ttyACM0（拔插会变） | /dev/serial/by-id/...（固定不变） |
| 串口权限 | 手动 sudo chmod | 脚本内自动执行 |
| 多模式切换 | 每次换模式要重新输一长串命令 | 选不同数字即可 |
| 相机识别 | 手动 ls /dev/video* + 猜测 | GUI 工具实时预览+自动生成配置 |
| 上手门槛 | 高（需记命令格式和参数） | 低（选数字即可） |

### 5.2 校准方式对比

**之前方案：**

```bash
# 每条臂需要手动输入完整命令
lerobot-calibrate \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left
```

问题：
- 端口用 /dev/ttyACM0，拔插后会变，容易搞错
- 参数多，容易拼错
- 每条臂都要输一遍长命令
- 串口权限需单独处理

**本方案：**

```bash
# 每条臂一个脚本，直接运行
python so101_cong_left.py
```

优势：
- 端口写死为 by-id 路径，拔插不影响
- 脚本内自动 sudo chmod
- 不用记任何参数
- 校准不匹配时按 Enter 即可写入

### 5.3 遥操作方式对比

**之前方案 — 单臂遥操作：**

```bash
lerobot-teleoperate \
  --teleop.type so101_leader \
  --teleop.port /dev/ttyACM1 \
  --teleop.id so101_zhu_left \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left \
  --fps 30
```

**之前方案 — 双臂遥操作：**

```bash
lerobot-teleoperate \
  --robot.type=bi_so_follower \
  --robot.left_arm_config.port=/dev/ttyACM0 \
  --robot.right_arm_config.port=/dev/ttyACM1 \
  --robot.id=bimanual_follower \
  --teleop.type=bi_so_leader \
  --teleop.left_arm_config.port=/dev/ttyACM2 \
  --teleop.right_arm_config.port=/dev/ttyACM3 \
  --teleop.id=bimanual_leader \
  --display_data=true \
  --fps 30
```

问题：
- 双臂命令极长，参数多且格式严格
- 端口号必须每次确认（拔插后会变）
- 一对多模式（单主控双从）无官方 CLI 支持
- 切换模式要重新输入整条命令

**本方案：**

```bash
python teleoperate.py
# 然后输入 1 / 2 / 3 选模式
```

优势：
- 一条命令覆盖三种模式
- 端口固定，不依赖 ttyACM 编号
- 一对多模式（之前无法实现）现可用
- 串口权限自动处理

### 5.4 功能覆盖对比

| 功能 | 之前方案 | 本方案 |
|---|---|---|
| 单臂遥操作（左） | ✅ CLI | ✅ 模式1 |
| 单臂遥操作（右） | ✅ CLI（需换参数） | ✅ 模式1 |
| 一对多遥操作 | ❌ 无 | ✅ 模式2 |
| 双臂独立遥操作 | ✅ bi_so_follower CLI | ✅ 模式3 |
| 带摄像头遥操作 | ✅ --robot.cameras 参数 | ❌ 暂未集成 |
| 数据录制 | ✅ lerobot-record | ❌ 暂未集成 |
| 数据回放 | ✅ lerobot-replay | ❌ 暂未集成 |
| 校准值写入电机 | ✅ Python 脚本片段 | ✅ 按 Enter 自动写入 |
| 标定数据迁移 | ✅ tar 打包文档 | ✅（沿用） |

### 5.5 架构差异

**之前方案**：直接调用 LeRobot 官方 CLI（`lerobot-teleoperate`、`lerobot-calibrate`），双臂模式使用 `bi_so_follower` / `bi_so_leader` 内置模块。

**本方案**：直接调用 LeRobot Python API（`SO101Follower`、`SO101Leader`），手动组合实现三种模式。不依赖 `bi_so_follower` / `bi_so_leader` 模块，而是在 Python 层面自行管理多个单臂实例的连接和控制循环。

```
本方案架构:
teleoperate.py
├── 模式1: SO101Leader + SO101Follower (一对一)
├── 模式2: SO101Leader + SO101Follower × 2 (一控多)
└── 模式3: SO101Leader × 2 + SO101Follower × 2 (双臂)
```

```
之前方案架构:
lerobot-teleoperate CLI
├── 单臂: --robot.type so101_follower + --teleop.type so101_leader
└── 双臂: --robot.type bi_so_follower + --teleop.type bi_so_leader
```

## 八、常见问题

### Q1: 校准时提示 "Mismatch between calibration values"
直接按 **Enter** 即可，程序会自动将已有校准文件写入电机。如需重新校准，按 **c**。

### Q2: 运行时 "Permission denied: /dev/ttyACM*"
脚本已内置 `sudo chmod 666`，但需要输入 sudo 密码。如不想每次输密码：
```bash
sudo usermod -aG dialout $USER
# 然后注销重新登录
```

### Q3: 拔插 USB 后脚本连不上臂
本方案使用 by-id 固定路径，正常情况拔插不影响。如果仍然连不上：
- 检查 USB 线是否松动
- 确认波特率已改为 500000
- 运行 `ls /dev/serial/by-id/` 确认设备仍在

### Q4: 电机过载保护 / "Overload error"
手动将机械臂移到安全位置，重新插拔 USB 上电，再运行对应校准脚本重写标定值。

### Q5: 升级 lerobot 后脚本报错
升级会覆盖波特率修复。需要重新修改 `DEFAULT_BAUDRATE = 500_000`，详见第二节环境准备。

### Q6: 需要 data recording 怎么办？
当前脚本不支持数据录制，请使用之前的 CLI 方案：
```bash
lerobot-record \
  --teleop.type so101_leader \
  --teleop.port /dev/ttyACM1 \
  --teleop.id so101_zhu_left \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left \
  --robot.cameras '{"top": {"index_or_path": "/dev/video2", "height": 480, "width": 640}}' \
  --fps 30 \
  --repo-id Ready321/你的数据集名 \
  --num-episodes 50 \
  --single-branch true
```
