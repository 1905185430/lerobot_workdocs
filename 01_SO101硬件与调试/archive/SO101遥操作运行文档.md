# SO-101 机械臂遥操作运行文档

> 适用于 LeRobot 0.5.1+ 版本（2026年4月更新）

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| LeRobot | >= 0.5.1 |
| Python | 3.12 |
| Conda 环境 | lerobot |
| GPU | NVIDIA (本地4060 Laptop / 远程PPIO 4090) |

## 2. 硬件连接

### 2.1 串口映射

| 设备 | 串口 | 说明 |
|------|------|------|
| 从臂 (Follower) | `/dev/ttyACM0` | 执行动作的机械臂 |
| 主臂 (Leader) | `/dev/ttyACM1` | 人手操控的主臂 |

### 2.2 摄像头

| 摄像头 | 设备路径 | 格式 |
|--------|----------|------|
| 顶部俯视 | `/dev/video8` | YUYV |
| 腕部 | `/dev/video10` | MJPG |

### 2.3 串口权限

```bash
# 方法1: 加入 dialout 组（需重新登录生效）
sudo usermod -aG dialout $USER
# 然后注销重新登录

# 方法2: 临时授权（每次重启后需重新执行）
sudo chmod 666 /dev/ttyACM0 /dev/ttyACM1
```

## 3. 波特率修复（重要！）

CH9101F 芯片在 1Mbps 下不可靠，必须改为 500000：

```bash
# 查找 feetech.py 路径（0.5.1 版本）
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")

# 修改 DEFAULT_BAUDRATE
sed -i 's/DEFAULT_BAUDRATE = 1_000_000/DEFAULT_BAUDRATE = 500_000/' "$FEETECH_PATH"

# 验证
grep "DEFAULT_BAUDRATE" "$FEETECH_PATH"
# 应输出: DEFAULT_BAUDRATE = 500_000
```

> ⚠️ 每次 pip install/upgrade lerobot 后需重新修改！

## 4. 标定

### 4.1 首次标定

标定文件存储在 `~/.cache/huggingface/lerobot/calibration/` 下：
- 从臂: `robots/so_follower/so101_cong_left.json`
- 主臂: `teleoperators/so_leader/so101_zhu_left.json`

首次使用需交互式标定（需在终端中手动操作）：

```bash
conda activate lerobot

# 标定从臂
lerobot-calibrate \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left

# 标定主臂
lerobot-calibrate \
  --robot.type so101_leader \
  --robot.port /dev/ttyACM1 \
  --robot.id so101_zhu_left
```

标定步骤（按终端提示操作）：
1. 将机械臂移到行程中间位置，按 Enter
2. 依次转动各关节通过完整运动范围
3. 按 Enter 完成记录

### 4.2 标定值写入电机

如果标定文件存在但提示 "Mismatch"，需手动将标定写入电机：

```bash
conda activate lerobot
python << 'EOF'
from lerobot.robots.so_follower.so_follower import SOFollower
from lerobot.robots.so_follower.config_so_follower import SOFollowerRobotConfig
from lerobot.teleoperators.so_leader.so_leader import SOLeader
from lerobot.teleoperators.so_leader.config_so_leader import SOLeaderTeleopConfig

# 写入从臂标定
follower = SOFollower(SOFollowerRobotConfig(id="so101_cong_left", port="/dev/ttyACM0"))
follower.bus.connect()
follower.bus.write_calibration(follower.calibration)
follower.configure()
follower.disconnect()
print("从臂标定写入完成")

# 写入主臂标定
leader = SOLeader(SOLeaderTeleopConfig(id="so101_zhu_left", port="/dev/ttyACM1"))
leader.bus.connect()
leader.bus.write_calibration(leader.calibration)
leader.configure()
leader.disconnect()
print("主臂标定写入完成")
EOF
```

## 5. 启动遥操作

### 5.1 标准遥操作命令

```bash
conda activate lerobot

lerobot-teleoperate \
  --teleop.type so101_leader \
  --teleop.port /dev/ttyACM1 \
  --teleop.id so101_zhu_left \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left \
  --fps 30
```

### 5.2 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `--teleop.type` | so101_leader | 主臂（人手操控）类型 |
| `--teleop.port` | /dev/ttyACM1 | 主臂串口 |
| `--teleop.id` | so101_zhu_left | 主臂标定ID |
| `--robot.type` | so101_follower | 从臂（执行臂）类型 |
| `--robot.port` | /dev/ttyACM0 | 从臂串口 |
| `--robot.id` | so101_cong_left | 从臂标定ID |
| `--fps` | 30 | 控制频率 |

### 5.3 带摄像头的遥操作

```bash
lerobot-teleoperate \
  --teleop.type so101_leader \
  --teleop.port /dev/ttyACM1 \
  --teleop.id so101_zhu_left \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left \
  --robot.cameras '{"top": {"index_or_path": "/dev/video8", "height": 480, "width": 640}, "wrist": {"index_or_path": "/dev/video10", "height": 480, "width": 640}}' \
  --display_data true \
  --fps 30
```

## 8. 常见问题

### Q1: Permission denied: /dev/ttyACM0
```bash
# 临时解决
sudo chmod 666 /dev/ttyACM0 /dev/ttyACM1
# 永久解决
sudo usermod -aG dialout $USER && 重新登录
```

### Q2: "There is no status packet!" 通信失败
- 检查 USB 线是否松动
- 确认波特率已改为 500000
- 重新插拔 USB 后重试

### Q3: "Mismatch between calibration values" 标定不匹配
- 参见 4.2 节手动写入标定值
- 或删除标定文件重新标定

### Q4: 升级 lerobot 后机械臂连不上
- 重新修改 feetech.py 的 DEFAULT_BAUDRATE
- 重新执行标定写入

### Q5: "Overload error" 电机过载保护
- 现象：某关节受力过大触发过载，导致 disconnect 失败、电机掉线
- 解决：手动将机械臂移到安全位置，重新插拔 USB 上电，再重写标定：
```bash
python << 'EOF'
from lerobot.robots.so_follower.so_follower import SOFollower
from lerobot.robots.so_follower.config_so_follower import SOFollowerRobotConfig
from lerobot.teleoperators.so_leader.so_leader import SOLeader
from lerobot.teleoperators.so_leader.config_so_leader import SOLeaderTeleopConfig

f = SOFollower(SOFollowerRobotConfig(id="so101_cong_left", port="/dev/ttyACM0"))
f.bus.connect(); f.bus.write_calibration(f.calibration); f.configure(); f.disconnect()
print("从臂 OK")

l = SOLeader(SOLeaderTeleopConfig(id="so101_zhu_left", port="/dev/ttyACM1"))
l.bus.connect(); l.bus.write_calibration(l.calibration); l.configure(); l.disconnect()
print("主臂 OK")
EOF
```

### Q6: Missing motor ID 6 / 电机掉线
- 原因：过载保护或 USB 通信异常导致电机掉线
- 解决：重新插拔机械臂 USB 线，执行 `sudo chmod 666 /dev/ttyACM*`，再重写标定

### Q7: PPIO 远程训练 "Output directory already exists"
- 解决方法1（继续训练）：加 `--resume=true`
- 解决方法2（重新训练）：换 `--output_dir=outputs/train/my_smolvla_v2`

## 9. LeRobot 0.4.4 → 0.5.1 API 变更速查

| 变更项 | 旧版 (0.4.4) | 新版 (0.5.1) |
|--------|-------------|-------------|
| motor 配置 | `{'name': (id, 'model')}` | `Motor(id, 'model', MotorNormMode.DEGREES)` |
| feetech 路径 | `lerobot.common.motors.feetech` | `lerobot.motors.feetech` |
| SO101 Robot | `SO101Robot` | `SOFollower` |
| SO101 Leader | 内置在 Robot | `SOLeader` (Teleoperator) |
| 遥操作命令 | `lerobot-control` | `lerobot-teleoperate` |
| 标定命令 | 内嵌于 connect | `lerobot-calibrate` 独立命令 |

## 10. 快速启动脚本

保存为 `~/teleop.sh`，一键启动遥操作：

```bash
#!/bin/bash
# SO-101 遥操作一键启动脚本

# 串口权限
sudo chmod 666 /dev/ttyACM0 /dev/ttyACM1 2>/dev/null

# 激活环境
source ~/anaconda3/etc/profile.d/conda.sh
conda activate lerobot

# 检查波特率
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")
if grep -q "DEFAULT_BAUDRATE = 1_000_000" "$FEETECH_PATH" 2>/dev/null; then
    echo "⚠️  检测到波特率为 1Mbps，正在修复为 500000..."
    sed -i 's/DEFAULT_BAUDRATE = 1_000_000/DEFAULT_BAUDRATE = 500_000/' "$FEETECH_PATH"
    echo "✅ 波特率已修复"
fi

# 启动遥操作
lerobot-teleoperate \
  --teleop.type so101_leader \
  --teleop.port /dev/ttyACM1 \
  --teleop.id so101_zhu_left \
  --robot.type so101_follower \
  --robot.port /dev/ttyACM0 \
  --robot.id so101_cong_left \
  --fps 30
```

```bash
chmod +x ~/teleop.sh
./teleop.sh
```
