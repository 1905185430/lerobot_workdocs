## 一、环境搭建与问题排查

### 1.1 lerobot-calibrate 命令找不到

**现象**：在 base 环境下运行 `lerobot-calibrate` 报 `command not found`

**原因**：LeRobot 安装在 `lerobot` conda 环境中，base 环境没有

**解决**：`conda activate lerobot` 切换环境

### 1.2 串口权限问题

**现象**：`PermissionError: [Errno 13] Permission denied: '/dev/ttyACM1'`

**临时解决**：`sudo chmod 666 /dev/ttyACM*`

**永久解决**：
```bash
sudo usermod -aG dialout $USER
```
重启后生效。

## 三、遥操作通信失败排查（核心问题）

### 3.1 现象

`lerobot-teleoperate` 运行后报错：
```
ConnectionError: Failed to sync read 'Present_Position' on ids=[1, 2, 3, 4, 5, 6] after 1 tries.
[TxRxResult] There is no status packet!
```
校准可通过，但遥操作时无法读取舵机位置。

### 3.2 排查过程

1. 校准能通过但遥操作失败，说明连接本身没问题
2. 用 scservo_sdk 逐波特率 ping 电机：

| 波特率  | 结果   |
|---------|--------|
| 9600    | OK     |
| 19200   | OK     |
| 38400   | OK     |
| 57600   | OK     |
| 115200  | OK     |
| 128000  | OK     |
| 250000  | OK     |
| 500000  | OK     |
| 1000000 | FAILED |

3. USB 转串口芯片为沁恒 CH9101F（1a86:55d3，cdc_acm 驱动），在 1Mbps 下通信不稳定
4. LeRobot 硬编码默认波特率 1000000，与实际不兼容

### 3.3 根因

LeRobot `feetech.py` 中 `DEFAULT_BAUDRATE = 1_000_000`，CH9101F 芯片不支持 1Mbps

### 3.4 修复

修改 `/home/xuan/lerobot/src/lerobot/motors/feetech/feetech.py`：
```python
# 原：DEFAULT_BAUDRATE = 1_000_000
DEFAULT_BAUDRATE = 500_000
```

editable 安装模式，修改立即生效。修改后需重新校准两个臂。

详细排查记录：`/home/xuan/le_xuan/SO101通信失败修复记录.md`

## 四、摄像头配置

### 4.1 设备识别

运行 `lerobot-find-cameras opencv` 检测结果：
- `/dev/video8`：顶部俯视摄像头（YUYV, 640x480@30fps）
- `/dev/video10`：腕部摄像头（MJPG, 640x480@30fps）
- video4/6 等为 Intel IPU6 内部处理节点，不可用，可忽略

### 4.2 遥操作命令（含双摄像头）

```bash
lerobot-teleoperate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=so101_cong_left \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=so101_zhu_left \
    --robot.cameras '{"top": {"type": "opencv", "index_or_path": "/dev/video8", "fps": 30, "width": 640, "height": 480}, "wrist": {"type": "opencv", "index_or_path": "/dev/video10", "fps": 30, "width": 640, "height": 480}}'
```

注意：摄像头配置字段名是 `index_or_path`，不是 `id`（会报 `DecodingError: The fields 'id' are not valid`）。

### 5.3 上传踩坑

**问题1**：录制后自动上传 HuggingFace 失败（SSL 错误），国内网络访问 huggingface.co 不稳定
- 解决：使用镜像 `export HF_ENDPOINT=https://hf-mirror.com`

**问题2**：LeRobot `push_to_hub` 整体上传卡在 99%，无法 commit
- 原因：`upload_large_folder` 模式下文件传到暂存区但最终 commit 步骤超时

**问题3**：`upload_folder` 整体上传同样卡在 99%
- 原因：大文件夹（含视频）一次性上传 commit 超时

**最终方案**：分批上传
1. 先传元数据和小文件（parquet、json），排除视频 — 成功
2. 顶视视频单独上传 — 成功
3. 腕部视频单独上传 — 成功

```bash
export HF_ENDPOINT=https://hf-mirror.com

# 小文件（排除视频）
python -c "
from huggingface_hub import HfApi
api = HfApi()
api.upload_folder(
    folder_path='/home/xuan/.cache/huggingface/lerobot/Dianrobot/vla_grab_redcube',
    repo_id='Ready321/vla_grab_redcube',
    repo_type='dataset',
    ignore_patterns=['videos/**'],
)
"

# 视频单独传（注意视频目录名是 observation.images.top/wrist，不是 top/wrist）
python -c "
from huggingface_hub import HfApi
api = HfApi()
api.upload_folder(
    folder_path='/home/xuan/.cache/huggingface/lerobot/Dianrobot/vla_grab_redcube/videos/observation.images.top',
    repo_id='Ready321/vla_grab_redcube',
    repo_type='dataset',
    path_in_repo='videos/observation.images.top',
)
"
```

## 六、SmolVLA 训练准备（PPIO GPU 服务器）

### 6.1 服务器选型

- 推荐：PPIO RTX 4090 (24GB) Spot 实例
- 镜像：PyTorch 2.x + CUDA 12.x
- 本地项目：`~/lerobot-vla-finetune/`（含环境检测、训练、上传脚本）

### 6.2 部署流程

```bash
# 1. 本地压缩项目
cd ~/lerobot-vla-finetune
tar -czvf lerobot-vla-finetune.tar.gz .

# 2. 上传到 PPIO 服务器
scp lerobot-vla-finetune.tar.gz root@PPIO-IP:/root/

# 3. SSH 到服务器
ssh root@PPIO-IP

# 4. 解压并安装
cd /root && tar -xzvf lerobot-vla-finetune.tar.gz
bash setup.sh

# 5. 设置镜像和登录
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli login
wandb login

# 6. 环境检测
python check_env.py

# 7. 开始训练
lerobot-train \
  --policy=smolvla \
  --dataset.repo_id=Ready321/vla_grab_redcube \
  --output_dir=/ppio-data/outputs/smolvla_grab_redcube \
  --batch_size=4 \
  --save_freq=500 \
  --wandb.enable=true
```

### 6.3 注意事项

- 输出目录用 `/ppio-data/`（持久存储，实例回收后不丢）
- Spot 实例可能被回收，`--save_freq=500` 勤存 checkpoint
- RTX 4090 24GB，batch_size 从 4 开始试
- 训练只读数据集，不涉及 feetech.py 波特率问题

## 七、注意事项汇总

1. ttyACM 编号每次插拔可能变化，使用前用 `ls /dev/ttyACM*` 确认
2. LeRobot 更新（git pull）可能覆盖 feetech.py 的波特率修改，需重新改
3. 上传数据集到 HF 镜像时，大文件夹整体上传可能卡住，建议分批上传
4. 视频目录名格式为 `observation.images.top`，不是简单的 `top`
5. 摄像头配置字段是 `index_or_path` 不是 `id`
6. PPIO 环境下缓存目录应设为 `/ppio-data/.cache/`（setup.sh 已自动处理）
