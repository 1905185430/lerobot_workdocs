## 2. SO-101 机械臂校准

从臂校准：
```bash
lerobot-calibrate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=so101_cong_left
```

主臂校准：
```bash
lerobot-calibrate \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=so101_zhu_left
```

校准时输入 `c` 重新校准，或直接回车使用已有校准文件。

## 3. 串口权限问题

**现象**：`PermissionError: [Errno 13] Permission denied: '/dev/ttyACM1'`

**临时解决**：`sudo chmod 666 /dev/ttyACM*`

**永久解决**：
```bash
sudo usermod -aG dialout $USER
```
重启后生效，不再需要 chmod。

## 4. 遥操作通信失败排查（核心问题）

**现象**：`lerobot-teleoperate` 运行时报 `ConnectionError: Failed to sync read 'Present_Position' ... There is no status packet!`

**排查过程**：
1. 校准能通过但遥操作失败，说明连接本身没问题
2. 用 scservo_sdk 逐波特率 ping 电机，发现 9600~500000 都能通，唯独 1000000 失败
3. USB 转串口芯片为沁恒 CH9101F（1a86:55d3，cdc_acm 驱动），在 1Mbps 下通信不稳定
4. LeRobot 硬编码默认波特率 1000000，与实际不兼容

**根因**：LeRobot `feetech.py` 中 `DEFAULT_BAUDRATE = 1_000_000`，CH9101F 芯片不支持 1Mbps

**修复**：修改 `/home/xuan/lerobot/src/lerobot/motors/feetech/feetech.py`
```python
# 原：DEFAULT_BAUDRATE = 1_000_000
DEFAULT_BAUDRATE = 500_000
```

editable 安装模式，修改立即生效。

**修复后**：重新校准两个臂（旧校准文件在错误波特率下生成），遥操作正常。

详细修复文档：`/home/xuan/le_xuan/SO101通信失败修复记录.md`

## 5. 摄像头识别

运行 `lerobot-find-cameras opencv` 检测到多个 video 设备，其中：
- `/dev/video8`：顶部俯视摄像头（YUYV, 640x480@30fps）
- `/dev/video10`：腕部摄像头（MJPG, 640x480@30fps）
- video4/6 等为 Intel IPU6 内部处理节点，不可用，可忽略

遥操作命令（含双摄像头）：
```bash
lerobot-teleoperate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=so101_cong_left \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=so101_zhu_left \
    --robot.cameras '{"top": {"type": "opencv", "index_or_path": "/dev/video8", "fps": 30, "width": 640, "height": 480}, "wrist": {"type": "opencv", "index_or_path": "/dev/video10", "fps": 30, "width": 640, "height": 480}}'
```

注意：摄像头配置字段名是 `index_or_path`，不是 `id`。

### 上传（通过 hf-mirror.com 镜像）

首次用 LeRobot 的 `push_to_hub` 和 `upload_folder` 整体上传，均在上传到 99% 后卡住不 commit，原因可能是镜像对大文件夹上传的 commit 机制不兼容。

最终方案：分批上传
1. 先传元数据和小文件（parquet、json），排除视频 — 成功
2. 顶视视频单独上传 — 成功
3. 腕部视频单独上传 — 成功

上传命令示例：
```bash
export HF_ENDPOINT=https://hf-mirror.com

# 小文件（排除视频）
python -c "
from huggingface_hub import HfApi
api = HfApi()
api.upload_folder(
    folder_path='数据集路径',
    repo_id='Ready321/vla_grab_redcube',
    repo_type='dataset',
    ignore_patterns=['videos/**'],
)
"

# 视频单独传
python -c "
from huggingface_hub import HfApi
api = HfApi()
api.upload_folder(
    folder_path='数据集路径/videos/observation.images.top',
    repo_id='Ready321/vla_grab_redcube',
    repo_type='dataset',
    path_in_repo='videos/observation.images.top',
)
"
```

## 注意事项备忘

1. ttyACM 编号每次插拔可能变化，使用前用 `ls /dev/ttyACM*` 确认
2. LeRobot 更新（git pull）可能覆盖 feetech.py 的波特率修改，需重新改
3. 上传数据集到 HF 镜像时，大文件夹整体上传可能卡住，建议分批上传
4. 摄像头配置字段是 `index_or_path` 不是 `id`
