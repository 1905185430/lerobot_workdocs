## 文档管理规范

### 命名规则

- 专题文档：`主题_副标题.md`，如 `SO101_MoveIt运动规划调研.md`
- 工作日志：`工作记录_YYYY-MM-DD.md` 或 `操作日志_YYYY-MM-DD.md`
- 报告类：`标题_YYYYMMDD.md`，如 `SmolVLA训练部署工作报告_20260415.md`

### 写文档的原则

1. **边做边记**：动手时同步记录，不要事后补（会遗漏细节）
2. **踩坑必录**：所有报错、解决过程、替代方案都要记
3. **结论先行**：文档开头先写核心结论/要点，再展开细节
4. **交叉引用**：文档间用相对路径互相链接
5. **标注日期**：每篇文档头部写明日期，方便追溯

### 分类规则

- 01：硬件连接、串口调试、舵机参数、遥操作 → 偏"手"的工作
- 02：训练脚本、数据集、模型部署 → 偏"脑"的工作
- 03：调研报告、方案对比、外部资料 → 偏"想"的工作
- 04：日志、周报、时间线 → 偏"记"的工作

拿不准放哪里就放 04_工作日志，后续再整理归档。
