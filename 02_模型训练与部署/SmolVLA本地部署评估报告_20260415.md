## 二、模型下载

### 2.1 微调模型权重

从 HuggingFace 镜像下载 `Ready321/my_smolvla`（~900MB）：

```bash
HF_ENDPOINT=https://hf-mirror.com hf download Ready321/my_smolvla --local-dir ~/下载/my_smolvla
```

> ⚠️ 注意：旧命令 `huggingface-cli download` 已废弃，使用 `hf download` 代替。
> 
> ⚠️ 不加 HF_ENDPOINT 时直连 huggingface.co 可能 SSL 报错（`UNEXPECTED_EOF_WHILE_READING`），加镜像后正常。

### 2.2 VLM 主干 SmolVLM2-500M-Video-Instruct

模型推理时需要额外下载 VLM 主干（~2GB），这是 SmolVLA 的视觉语言部分，不在微调权重里。

**方案一：镜像下载（推荐）**
```bash
HF_ENDPOINT=https://hf-mirror.com hf download HuggingFaceTB/SmolVLM2-500M-Video-Instruct
```

**方案二：从 Jetson 局域网拷贝（更快）**
```bash
# Jetson 上已有完整缓存
scp -r orin-1@192.168.18.149:/home/orin-1/.cache/huggingface/hub/models--HuggingFaceTB--SmolVLM2-500M-Video-Instruct \
  ~/.cache/huggingface/hub/
```

> 局域网传输约 24MB/s，1.9GB 约 1-2 分钟；镜像下载约 2-5MB/s，需要 10+ 分钟。

## 三、环境兼容性修复（关键！）

本机环境 `lerobot-seeed`（0.4.3）与 PPIO 上训练用的 0.4.4 存在多处不兼容，需逐一修复：

### 3.1 删除 config.json 中的 `use_peft` 字段

**现象**：`lerobot-record` 加载模型时报 `TypeError: __init__() got an unexpected keyword argument 'use_peft'`

**原因**：0.5.x 训练时将 `use_peft` 写入 config.json，但 0.4.4 的 SmolVLAConfig 不认识该字段。

**解决**：
```bash
# 方法一：手动删除
python3 -c "
import json
with open('~/下载/my_smolvla/config.json') as f:
    d = json.load(f)
d.pop('use_peft', None)
with open('~/下载/my_smolvla/config.json', 'w') as f:
    json.dump(d, f, indent=2)
"
```

### 3.2 SmolVLA 注册缺失（最关键的 bug！）

**现象**：`lerobot-record` 启动后 policy 加载静默失败，机械臂空转，日志不断打印：
```
No policy or teleoperator provided, skipping action generation.
```

**原因**：`lerobot-seeed` 的 `lerobot_record.py` 通过 `from lerobot.configs.policies import PreTrainedConfig` 导入基类时，由于循环导入，`policies/__init__.py` 中的 `from .smolvla.configuration_smolvla import SmolVLAConfig` 未被执行，导致 `smolvla` 未注册到 `PreTrainedConfig._choice_registry`。

**验证方法**：
```python
from lerobot.configs.policies import PreTrainedConfig
print('smolvla' in PreTrainedConfig._choice_registry)  # False ← 有bug
from lerobot.policies.act.configuration_act import ACTConfig
print('smolvla' in PreTrainedConfig._choice_registry)  # True ← 导入任意具体Config后触发
```

**解决**：在 `lerobot_record.py` 中显式导入 SmolVLAConfig：

```python
# /home/xuan/lerobot-seeed/src/lerobot/scripts/lerobot_record.py
# 在 from lerobot.configs.policies import PreTrainedConfig 后面加一行：
from lerobot.policies.smolvla.configuration_smolvla import SmolVLAConfig  # noqa: F401 — ensure smolvla is registered
```

### 3.3 policy.path 不能用 `~`

**现象**：`HFValidationError: Repo id must use alphanumeric chars... '~/my_smolvla'`

**原因**：`~` 未被 shell 展开，lerobot 将其当作 HuggingFace repo_id 解析。

**解决**：用绝对路径 `--policy.path=/home/xuan/下载/my_smolvla`

### 3.4 摄像头名称必须匹配训练时

**现象**：`ValueError: Feature mismatch... Missing: camera1, camera2, camera3. Extra: front, side`

**原因**：推理时摄像头名（front/side）与训练时（camera1/camera2）不一致。

**解决**：直接在 `--robot.cameras` 中使用 `camera1`、`camera2` 命名，无需 rename_map。

### 3.5 `num2words` 缺失

**现象**：`ImportError: Package 'num2words' is required to run SmolVLM processor`

**解决**：
```bash
pip install num2words
```

## 四、本机硬件识别

### 4.1 摄像头设备

```bash
ls /dev/video*
```

本机可用摄像头及格式：

| 设备 | 格式 | 分辨率 | 用途 |
|------|------|--------|------|
| /dev/video0 | YUYV | 480x640 | — |
| /dev/video2 | MJPG | 480x640 | camera1（俯视） |
| /dev/video6 | NV12 | 400x640 | — |
| /dev/video8 | YV12 | 400x640 | — |
| /dev/video10 | MJPG | 480x640 | camera2（腕部） |

> ⚠️ 本机和 Jetson 的摄像头设备号不同！Jetson 上是 video0 和 video6，本机是 video2 和 video10。
> 
> ⚠️ video8 在本机不支持 MJPG（YV12格式），不能加 fourcc: "MJPG"。
> 
> ⚠️ video2 支持 MJPG 但在 lerobot 中加 fourcc: "MJPG" 时会报 `!buf.empty()` 错误，去掉 fourcc 让 OpenCV 自动选择即可。

### 4.2 机械臂串口

| 设备 | 用途 |
|------|------|
| /dev/ttyACM0 | 从臂 so101_cong_left (follower) |
| /dev/ttyACM1 | 主臂 so101_zhu_left (leader) |

> ⚠️ Motor 6（夹爪）检测失败，当前从臂为 5 自由度版本（5个电机 ID 1-5）。需要调整配置或检查夹爪连接。

## 五、评估命令

### 5.1 最终成功的命令

```bash
conda activate lerobot-seeed

rm -rf ~/.cache/huggingface/lerobot/Ready321/eval_grab_redcube_test

HF_ENDPOINT=https://hf-mirror.com lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.id=so101_cong_left \
  --robot.cameras='{ camera1: {type: opencv, index_or_path: 2, width: 640, height: 480, fps: 30}, camera2: {type: opencv, index_or_path: 10, width: 640, height: 480, fps: 30} }' \
  --dataset.single_task="Grab the red cube to the white cup" \
  --dataset.repo_id=Ready321/eval_grab_redcube_test \
  --dataset.episode_time_s=50 \
  --dataset.num_episodes=10 \
  --dataset.push_to_hub=false \
  --policy.path=/home/xuan/下载/my_smolvla \
  --policy.empty_cameras=1 \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM1 \
  --teleop.id=so101_zhu_left
```

### 5.2 参数详解

| 参数 | 值 | 说明 |
|------|----|------|
| `HF_ENDPOINT` | https://hf-mirror.com | 国内镜像，huggingface.co 访问不稳定 |
| `--robot.type` | so101_follower | SO-101 从臂（被控端） |
| `--robot.port` | /dev/ttyACM0 | 从臂串口 |
| `--robot.id` | so101_cong_left | 机器人 ID，对应校准文件 |
| `--robot.cameras` | camera1(2), camera2(10) | 摄像头名必须匹配训练时的 camera1/camera2 |
| `index_or_path` | 2, 10 | 摄像头设备号，本机实测可用 |
| `--dataset.single_task` | Grab the red cube... | 任务描述，必须与训练一致 |
| `--dataset.repo_id` | Ready321/eval_grab_redcube_test | 评估数据保存路径 |
| `--dataset.episode_time_s` | 50 | 每个 episode 最长 50 秒 |
| `--dataset.num_episodes` | 10 | 共 10 个 episode |
| `--dataset.push_to_hub` | false | 不推送到 HF |
| `--policy.path` | /home/xuan/下载/my_smolvla | 模型权重绝对路径（不能用~） |
| `--policy.empty_cameras` | 1 | 只有 2 个摄像头，第 3 个空帧填充 |
| `--teleop.type` | so101_leader | 主臂（用于复位阶段手动控制） |
| `--teleop.port` | /dev/ttyACM1 | 主臂串口 |
| `--teleop.id` | so101_zhu_left | 主臂 ID |

## 六、评估流程说明

### 6.1 运行阶段

1. **模型加载**：加载 SmolVLA policy + VLM 主干（SmolVLM2-500M），占用 GPU 约 7.2GB/8GB
2. **硬件连接**：连接从臂、摄像头、主臂
3. **Episode 录制**：policy 驱动从臂执行动作（50秒/episode）
4. **环境复位**：episode 间有 60 秒复位时间，通过主臂手动操控从臂回到初始位置
5. **重复**：直到 10 个 episode 完成


RTX 4060 Laptop (8GB) 刚好够用，但余量很少。建议关闭其他占用显存的程序。
