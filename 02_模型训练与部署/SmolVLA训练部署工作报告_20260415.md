## 二、遇到的困难与解决方案

### 困难1：缺少 datasets 依赖

**现象：**
```
ImportError: 'datasets' is required but not installed.
```

**原因：** 服务器上 lerobot 环境未安装 `lerobot[dataset]` 额外依赖。

**解决：** 在 lerobot conda 环境内执行 `pip install 'lerobot[dataset]'`

**踩坑：** 第一次装到了系统 Python（`/usr/bin/python3.10`）而非 conda 环境，因为没先激活环境。后续所有操作都确保先 `source conda.sh && conda activate lerobot`。

---

### 困难2：缺少 wandb 模块

**现象：**
```
ModuleNotFoundError: No module named 'wandb'
```

**原因：** `lerobot[dataset]` 的依赖列表中不包含 wandb，需单独安装。

**解决：** `pip install wandb`，然后 `wandb login <token>` 登录。

---

### 困难3：policy.repo_id 参数缺失

**现象：**
```
ValueError: 'policy.repo_id' argument missing. Please specify it to push the model to the hub.
```

**原因：** lerobot 0.5.2 要求必须指定 `--policy.repo_id`，用于训练完成后推送模型到 HF Hub。

**解决：** 添加 `--policy.repo_id=Ready321/my_smolvla`

---

### 困难4：数据集版本标签缺失（核心问题）

**现象：**
```
RevisionNotFoundError: Your dataset must be tagged with a codebase version.
```

**原因：** 数据集是用 lerobot 0.4.4 上传的，0.4.4 不会自动给 HF 仓库打 git tag。而 0.5.2 通过 git tag 来定位数据集版本，找不到就报错。

**解决：** 用 HfApi 给数据集打版本标签：
```python
from huggingface_hub import HfApi
api = HfApi(token='<HF_TOKEN>')
api.create_tag('Ready321/vla_grab_redcube', tag='v0.5.2', repo_type='dataset')
```

**踩坑：** 服务器上未登录 HF，第一个 token 已失效，需要嘉璇提供新 token。

---

### 困难5：数据集 codebase_version 与 lerobot 版本不匹配（核心问题）

**现象：**
```
BackwardCompatibilityError / NotImplementedError: Contact the maintainer on Discord.
```

**原因：** 数据集 info.json 中 `codebase_version: v3.0`（0.4.x 的格式），而服务器 lerobot 0.5.2 期望 `v0.5.2`。两者 major 版本不匹配（3 vs 0），lerobot 认为大版本不兼容。

**根本原因：** lerobot 的版本规范在新版中发生了变化——旧版用 `v3.0` 表示数据格式版本，新版改为与 codebase 版本对齐的 `v0.5.2` 语义化版本格式。

**解决：** 更新数据集 info.json 中的 `codebase_version` 为 `v0.5.2`，同时补充 0.5.2 格式所需的新字段：
- `video_info` 替代旧的 `info` 字段
- 各 feature 添加 `fps` 字段
- 添加 `next.done` feature

然后重新上传 info.json 到 HF：
```python
upload_file(
    path_or_fileobj='/tmp/info.json',
    path_in_repo='meta/info.json',
    repo_id='Ready321/vla_grab_redcube',
    repo_type='dataset',
)
```

### 困难6：特征名不匹配

**现象：**
```
ValueError: Feature mismatch between dataset/environment and policy config.
- Missing features: ['observation.images.camera1', 'observation.images.camera2', 'observation.images.camera3']
- Extra features: ['observation.images.top', 'observation.images.wrist']
```

**原因：** SmolVLA 模型默认期望摄像头命名为 `camera1/camera2/camera3`，但我们的数据集用的是 `top/wrist`（顶部俯视 + 腕部）。

**初始尝试（失败）：** 使用 `--rename_map` 命令行参数，但 draccus（lerobot 的配置解析库）无法从 CLI 解析 `dict[str, str]` 类型，各种格式（JSON、key:value、key=value）均失败。

**最终解决：** 不用 rename_map，直接在 Python 脚本中构建 SmolVLAConfig，覆盖 `input_features` 使其匹配数据集的 `top/wrist` 命名：
```python
policy_cfg.input_features = {
    'observation.images.top': PolicyFeature(type=FeatureType.VISUAL, shape=(3, 480, 640)),
    'observation.images.wrist': PolicyFeature(type=FeatureType.VISUAL, shape=(3, 480, 640)),
    'observation.state': PolicyFeature(type=FeatureType.STATE, shape=(6,)),
}
```

---

### 困难7：torchcodec 加载失败

**现象：**
```
OSError: Could not load this library: libtorchcodec_core5.so
OSError: libavutil.so.56: cannot open shared object file
```

**原因：** 服务器缺少 FFmpeg 开发库，torchcodec 无法加载视频解码器。

**解决：** `apt install ffmpeg libavutil-dev libavcodec-dev libavformat-dev libswscale-dev`

---

### 困难8：PreTrainedConfig 是抽象类

**现象：**
```
TypeError: Can't instantiate abstract class PreTrainedConfig
```

**原因：** 最初尝试用 `PreTrainedConfig` 构建 policy 配置，但它是抽象类，有未实现的抽象方法。

**解决：** 使用具体的 `SmolVLAConfig` 类替代。

---

### 困难9：draccus CLI 参数解析问题

**现象：** `--policy.path`、`--rename_map` 等参数通过命令行传递时反复失败。

**根本原因：** lerobot 使用 draccus 库解析 CLI，`--policy.path` 是 lerobot 的 `@wrap` 装饰器特殊处理的（从 CLI 中移除，后续单独加载），而 `dict` 类型参数无法从 CLI 传入。

**最终解决：** 完全放弃 CLI 方式，编写 Python 训练脚本 `run_train.py`，程序化构建配置对象，直接调用 `train(cfg)`。

## 六、经验教训

1. **永远先确认 conda 环境** — 直接 `pip install` 可能装到系统 Python
2. **HF 数据集必须打 version tag** — lerobot 0.5.x 强制要求，旧版上传的数据集需手动补打
3. **codebase_version 必须匹配** — 新旧版 lerobot 的版本规范不同，升级后需更新数据集元数据
4. **特征命名要统一** — 采集时用的摄像头名（top/wrist）需与模型期望的（camera1/2/3）映射
5. **复杂参数用配置文件** — 命令行传 JSON/dict 在 SSH 多层转义中容易出错
6. **HF token 会过期** — 备好最新 token，服务器端需重新登录
