| 臂名称 | 角色 | by-id 路径（固定） |
|--------|------|-------------------|
| so101_cong_left | 从臂（左） | `/dev/serial/by-id/usb-1a86_USB_Single_Serial_5B42073876-if00` |

## 3. 实验一：环境搭建与验证

**目标**：安装 LeRobot 并确保机械臂通信正常。

**预计时间**：30 分钟

### 步骤 1：激活 Conda 环境

```bash
conda activate lerobot
```

验证版本：

```bash
python -c "import lerobot; print(lerobot.__version__)"
# 期望输出: 0.5.1
```

> 如果提示找不到 lerobot 环境，请先参考 [06_环境安装记录/Seeed版LeRobot安装记录.md](06_环境安装记录/Seeed版LeRobot安装记录.md) 安装。

### 步骤 2：检查波特率（必须！）

SO-101 使用的 CH9101F USB 芯片在默认 1Mbps 下通信不可靠，**必须修改为 500000**：

```bash
FEETECH_PATH=$(python -c "import lerobot.motors.feetech.feetech as m; print(m.__file__)")
grep "DEFAULT_BAUDRATE" "$FEETECH_PATH"
```

- 如果输出 `DEFAULT_BAUDRATE = 500_000` → 已修复，跳过
- 如果输出 `DEFAULT_BAUDRATE = 1_000_000` → 需要修复：

```bash
sed -i 's/DEFAULT_BAUDRATE = 1_000_000/DEFAULT_BAUDRATE = 500_000/' "$FEETECH_PATH"
```

> ⚠️ **每次** `pip install` / `git pull` / `conda update lerobot` 后都会覆盖此修改，需重新执行！

### 步骤 3：连接硬件

1. 将 4 条臂的 USB-C 线接入电脑
2. 接通 4 个电源适配器
3. 用桌夹固定臂底座

确认系统识别设备：

```bash
ls /dev/serial/by-id/usb-1a86*
```

期望输出 4 行 by-id 路径。如果少于 4 行，检查 USB 线是否插紧。

### 步骤 4：串口权限

```bash
# 临时授权（每次重启后需重新执行）
sudo chmod 666 /dev/ttyACM* /dev/ttyUSB* 2>/dev/null

# 或永久授权（推荐，执行后需重新登录）
sudo usermod -aG dialout $USER
```

### 步骤 5：验证通信

```bash
python -c "
from lerobot.robots.so_follower import SO101Follower, SO101FollowerConfig
cfg = SO101FollowerConfig(
    port='/dev/serial/by-id/usb-1a86_USB_Single_Serial_5B42073876-if00',
    id='so101_cong_left',
)
robot = SO101Follower(cfg)
robot.connect()
obs = robot.get_observation()
print('通信成功！关节角度：', {k: round(v, 2) for k, v in obs.items() if not isinstance(v, object) or hasattr(v, '__len__') and len(str(v)) < 20})
robot.disconnect()
"
```

如果能打印出关节角度数值，说明通信正常。

### 实验一检查点

- [ ] LeRobot 0.5.1 环境可用
- [ ] 波特率已改为 500000
- [ ] 4 条臂均被系统识别
- [ ] 串口权限正常
- [ ] 能读取关节角度

## 4. 实验二：机械臂校准

**目标**：理解校准原理，完成所有臂的校准。

**预计时间**：20 分钟（首次）/ 5 分钟（后续按 Enter 即可）

### 背景知识

SO-101 的电机输出的是原始编码值（0-4095），校准的作用是建立编码值与物理角度的映射关系。校准数据存储在：

```
~/.cache/huggingface/lerobot/calibration/
├── robots/so_follower/
│   ├── so101_cong_left.json
│   └── ...
└── teleoperators/so_leader/
    ├── so101_zhu_left.json
    └── ...
```

### 何时需要校准

| 场景 | 是否需要 |
|------|---------|
| 首次使用新臂 | 是 |
| 拔插 USB 后 | 否 |
| 波特率修改后 | 是 |
| 更换电机后 | 是 |
| 换电脑（同臂迁移） | 否（复制校准文件即可） |
| 提示 "Mismatch between calibration values" | 否（按 Enter 写入） |

### 步骤 1：校准从臂

```bash
conda activate lerobot

# 左从臂
python 01_校准/calibrate_follower_left.py
# 右从臂
python 01_校准/calibrate_follower_right.py
```

**交互流程：**

1. 脚本自动执行 `sudo chmod 666`（可能需要输入密码）
2. 连接臂成功后，进入校准模式
3. 终端提示将臂移到"中间位置"——将各关节调整到大致居中位置
4. 按 **Enter** 确认
5. 终端提示转动各关节——手动将每个关节缓慢转动过完整范围
6. 按 **Enter** 完成校准
7. 校准数据自动保存，臂断开连接

### 步骤 2：校准主臂

```bash
# 左主臂
python 01_校准/calibrate_leader_left.py
# 右主臂
python 01_校准/calibrate_leader_right.py
```

流程与从臂相同。

### 步骤 3：验证校准数据

```bash
ls ~/.cache/huggingface/lerobot/calibration/robots/so_follower/
ls ~/.cache/huggingface/lerobot/calibration/teleoperators/so_leader/
```

应看到 4 个 `.json` 文件（每条臂一个）。

### 校准不匹配的处理

启动遥操作时如果提示 "Mismatch between calibration values"：

- **按 Enter** → 用已有校准文件写入电机（推荐，秒完成）
- **按 c** → 重新交互式校准

### 实验二检查点

- [ ] 4 条臂全部校准完成
- [ ] 校准 JSON 文件存在
- [ ] 理解按 Enter 写入 vs 重新校准的区别

## 5. 实验三：遥操作控制

**目标**：实现主臂对从臂的实时遥控，掌握三种遥操作模式。

**预计时间**：15 分钟

### 背景知识

SO-101 遥操作采用"主从"架构：

- **主臂（Leader）**：人手握持，读取人手动作
- **从臂（Follower）**：执行端，跟随主臂运动

主臂读取各关节角度 → 发送给从臂 → 从臂运动到目标位置，循环执行（30Hz）。

### 步骤 1：单臂遥操作

```bash
conda activate lerobot
python 02_遥操作/teleoperate.py
```

选择 **模式 1（一对一遥操作）**，再选择 **左臂对**。

终端输出：

```
✅ 连接成功！按 Ctrl+C 退出遥操作。
```

此时握住左主臂移动，左从臂应同步跟随。

> 如果提示 "Mismatch between calibration values"，按 Enter 写入即可。

**练习**：尝试以下动作，感受主从跟随的延迟和精度：
1. 缓慢转动肩关节
2. 快速摆动手腕
3. 轻柔地开合夹爪

按 **Ctrl+C** 退出。

### 步骤 2：一对多遥操作（进阶）

再次运行脚本，选择 **模式 2**。

右主臂同时控制左、右两个从臂做同步运动。适用于双臂对称任务。

### 步骤 3：双臂遥操作（进阶）

选择 **模式 3**。

左主臂 → 左从臂，右主臂 → 右从臂，两条臂独立控制。适用于双臂协同任务（如交接物体）。

### 步骤 4：带摄像头的遥操作

```bash
python 02_遥操作/teleoperate_2cam.py   # 双摄像头
python 02_遥操作/teleoperate_3cam.py   # 三摄像头
```

启动后 Rerun 可视化窗口会自动弹出，实时显示摄像头画面。

> 如果 Rerun 窗口没有弹出，请确保通过 `conda activate lerobot` 启动（不要用 python 绝对路径），这样 PATH 中才包含 rerun 可执行文件。

### 实验三检查点

- [ ] 单臂遥操作正常，从臂实时跟随
- [ ] 理解三种模式的适用场景
- [ ] 摄像头画面在 Rerun 中显示正常
- [ ] 能通过 Ctrl+C 安全退出

## 6. 实验四：摄像头配置与测试

**目标**：确认摄像头索引号，验证画面采集正常。

**预计时间**：15 分钟

### 背景知识

Linux 下摄像头以 `/dev/videoN` 形式出现，N 为索引号。注意：

- 每个物理摄像头通常产生 2 个 video 设备（一个是图像流，一个是 metadata）
- video 编号在拔插后会变化
- 同型号摄像头 Serial 相同时，by-id 无法区分

### 步骤 1：查看所有视频设备

```bash
ls /dev/video*
```

或使用详细工具：

```bash
v4l2-ctl --list-devices
```

记录每个摄像头的 video 编号。

### 步骤 2：使用 GUI 工具识别摄像头（推荐）

```bash
pip install PySimpleGUI opencv-python   # 首次安装
python 04_摄像头工具/camera_identifier.py
```

操作流程：
1. 左侧列表选择设备 → 右侧实时预览画面
2. 确认每个摄像头对应的视角（俯视/腕部/侧面）
3. 输入名称 → 保存配置

### 步骤 3：快速画面测试

```bash
python 04_摄像头工具/camera_capture.py
```

逐个摄像头拍照保存到 `04_摄像头工具/outputs/`，确认画面正常。

### 步骤 4：安装 udev 规则（可选，推荐）

如果生成了 udev 规则文件：

```bash
sudo cp 04_摄像头工具/99-so101-cameras.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger
```

安装后摄像头设备名固定，不再受 video 编号变化影响。

### 实验四检查点

- [ ] 记录了每个摄像头的 video 编号
- [ ] 确认了哪个编号对应哪个视角（俯视/腕部/侧面）
- [ ] 画面采集正常，无黑屏/花屏
- [ ] 了解 metadata 设备（video N+1）不可用于读帧

## 7. 实验五：数据采集

**目标**：完成一次完整的遥操作数据采集，理解数据集的结构。

**预计时间**：60 分钟

### 背景知识

LeRobot 数据采集的核心概念：

| 概念 | 说明 |
|------|------|
| Episode | 一次完整的任务执行过程（如：拿起红色方块 → 放下） |
| Frame | 一帧数据，包含观测（关节角度+图像）和动作 |
| FPS | 帧率，控制采集频率，默认 30Hz |
| Task | 任务描述标签，用于训练时区分不同任务 |
| Dataset | 所有 episode 的集合，存储为视频+parquet 格式 |

数据集本地路径：`~/.cache/huggingface/lerobot/<repo_id>/`

### 步骤 1：准备采集场景

1. 将红色方块放在桌面固定位置
2. 调整从臂到初始姿态
3. 确认摄像头画面正常（参照实验四）
4. 确保 conda 环境已激活：

```bash
conda activate lerobot
```

### 步骤 2：修改采集参数（如需）

打开 `03_数据采集/record_so101_2cam.py` 或 `record_so101_3cam.py`，检查/修改配置区：

```python
NUM_EPISODES = 50                  # 采集 episode 总数
FPS = 30                           # 帧率
EPISODE_TIME_SEC = 60              # 每个 episode 最长录制时间（秒）
RESET_TIME_SEC = 10                # 重置环境时间（秒）
TASK_DESCRIPTION = "grab_red_cube" # 任务描述
DATASET_REPO_ID = "Ready321/so101_grab_redcube_3cam"  # HF 数据集名

# 摄像头索引（必须与实验四确认的一致！）
TOP_CAM = 10       # 俯视
WRIST_CAM = 2      # 腕部
SIDE_CAM = 18      # 侧面（仅 3cam 脚本）
```

> ⚠️ 摄像头索引号必须与实际连接一致，否则会报错或采集黑屏！

### 步骤 3：启动采集

**双摄像头：**

```bash
python 03_数据采集/record_so101_2cam.py
```

**三摄像头：**

```bash
python 03_数据采集/record_so101_3cam.py
```

**自定义数据集名：**

```bash
python 03_数据采集/record_so101_3cam.py --name grab_blue_cube
# 实际 repo_id: Ready321/grab_blue_cube
```

**覆盖已有数据集：**

```bash
python 03_数据采集/record_so101_3cam.py --overwrite
```

> 如果不加 --overwrite 且数据集目录已存在，会自动追加时间戳后缀（如 `so101_grab_redcube_3cam_20260417_143000`），旧数据不会被删除。

### 步骤 4：执行录制

脚本启动后会依次录制每个 episode。录制过程中的操作：

**每个 Episode 的流程：**

```
1. 终端显示 "录制 Episode N / 50"
2. 操控主臂执行任务（如抓取红色方块）
3. 完成后：
   - 按 右箭头 → 提前结束当前 episode（数据保留）
   - 按 左箭头 → 丢弃当前 episode，重新录制
   - 什么都不按 → 等待 EPISODE_TIME_SEC 秒后自动结束
4. 终端显示 "重置环境中..." → 将物体放回初始位置
5. 等待 RESET_TIME_SEC 秒
6. 自动进入下一个 episode
```

**提前结束所有录制：** 按 **ESC** 键。

### 步骤 5：采集操作要点

为了保证数据质量，请注意：

1. **起始位置一致**：每个 episode 开始时，物体和从臂应处于相同的初始位置
2. **动作流畅**：避免突然的急停或回退，保持运动轨迹自然平滑
3. **任务完成判定**：任务完成后按右箭头提前结束，不必等满 60 秒
4. **失败重录**：如果某次操作失败（掉了、没抓住），按左箭头重录
5. **重置要充分**：重置阶段将物体放回原位，环境恢复到初始状态

### 步骤 6：查看采集结果

采集完成后，终端输出：

```
录制结束，正在保存...
共录制 50 个 episode
数据集本地路径：~/.cache/huggingface/lerobot/Ready321/so101_grab_redcube_3cam
```

查看数据集结构：

```bash
ls ~/.cache/huggingface/lerobot/Ready321/so101_grab_redcube_3cam/
```

典型内容：

```
meta/
  info.json           ← 数据集元信息（FPS、特征定义等）
  stats.json          ← 统计信息（均值、方差等）
  tasks.jsonl         ← 任务描述
data/
  train-00000-of-00001.parquet   ← 动作+观测数值数据
videos/
  observation.images.top/        ← 俯视摄像头视频
    episode_000000.mp4
  observation.images.wrist/      ← 腕部摄像头视频
    episode_000000.mp4
  observation.images.side/       ← 侧面摄像头视频（3cam）
    episode_000000.mp4
```

## 8. 常见问题排查

### 8.1 通信失败

| 症状 | 原因 | 解决 |
|------|------|------|
| "There is no status packet!" | 波特率 1Mbps 不可靠 | 改为 500000（见实验一步骤2） |
| "Permission denied: /dev/ttyACM*" | 串口权限不足 | `sudo chmod 666 /dev/ttyACM*` |
| 连接后无响应 | USB 线松动 | 重新插拔 USB |
| 升级 lerobot 后连不上 | 波特率被还原 | 重新修改 DEFAULT_BAUDRATE |

### 8.2 校准问题

| 症状 | 解决 |
|------|------|
| "Mismatch between calibration values" | 按 Enter 写入已有校准文件 |
| 校准文件找不到 | 确认 robot.id 正确；双臂模式用 bimanual_ 前缀 |
| 需要重新校准 | 按 c 进入交互式校准，或删除校准 JSON 后重跑 |

### 8.3 摄像头问题

| 症状 | 解决 |
|------|------|
| 画面黑屏 | video 编号不对，用 v4l2-ctl 重新确认 |
| buf.empty 报错 | 不要在 OpenCVCameraConfig 中指定 fourcc 参数 |
| USB 带宽不足丢帧 | 降低分辨率到 320x240 或降低 FPS |
| video 编号变了 | 安装 udev 规则固定设备名 |

### 8.4 数据采集问题

| 症状 | 解决 |
|------|------|
| FileExistsError | 使用 `--overwrite` 或 `--name` 参数，或脚本会自动加时间戳 |
| Rerun 窗口不弹出 | 通过 `conda activate lerobot` 启动，不用 python 绝对路径 |
| 采集速度慢于目标 FPS | 检查摄像头是否支持目标帧率，或降低分辨率 |

### 8.5 电机问题

| 症状 | 原因 | 解决 |
|------|------|------|
| "Overload error" | 关节受力过大 | 手动移到安全位 → 插拔 USB → 重写校准 |
| Missing motor ID 6 | 过载掉线 | 插拔 USB → chmod → 重写校准 |
| 一条臂掉线 | USB 问题 | 只需恢复掉线臂，不影响另一条 |
